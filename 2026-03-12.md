**Super Keyword**

The super keyword in Kotlin is used to refer to the parent (superclass) implementation of a property, function, or constructor. It helps a subclass access or call members of its parent class. 

When a child class overrides a function, super lets you call the original implementation from the parent class.

            open class Animal {
                open fun sound() {
                    println("Animal makes a sound")
                }
            }
            
            class Dog : Animal() {
                override fun sound() {
                    super.sound()   // calls parent method
                    println("Dog barks")
                }
            }
            
            fun main() {
                val d = Dog()
                d.sound()
            }
            
super can also access a property defined in the parent class.


      open class Vehicle {
          open val speed = 100
      }
      
      class Car : Vehicle() {
          override val speed = 150
      
          fun showSpeed() {
              println("Vehicle speed: " + super.speed)
              println("Car speed: " + speed)
          }
      }

**Encapsulation:**

Encapsulation is the principle of hiding internal workings of a class.

Encapsulation  is the process of hiding data inside a class and providing controlled access using methods, getters, setters, and access modifiers.

Only show what the others classes need

**Advantages of Encapsulation**

Protects data 


Improves code security


Makes code easier to maintain


Prevents accidental changes

**Example:**

            
            fun main() {
                val account=BankAccount()
                account.deposit(500.0)
                println(account.getBalance())
            }
            class BankAccount{
                
                private var balance:Double=0.0
                
                fun deposit(amount : Double){
                    if(amount>0){
                        balance += amount
                    }
                }
                fun getBalance():Double{
                    return balance
                }
            }



Explanation

balance is private, so it cannot be accessed directly outside the class.


Access is allowed through:


deposit()


getBalance()


This protects the balance from invalid changes.

**Abstraction**

Abstraction is a concept that focuses on hiding complex implementation details and exposing only the necessary functionality.

Abstraction is implemented using:

Abstract Classes


Interfaces

**Abstraction Using Abstract Class**

An abstract class is a class that cannot be instantiated (cannot create objects) and may contain:
Abstract functions → functions without implementation


Normal functions → functions with implementation


**Syntax**


      abstract class ClassName {
          abstract fun functionName()
      }


---------------------------------------------



            abstract class Animal {
            
                abstract fun makeSound()
            
                fun sleep() {
                    println("Animal is sleeping")
                }
            }
            
            class Dog : Animal() {
            
                override fun makeSound() {
                    println("Dog barks")
                }
            }
            
            fun main() {
                val dog = Dog()
                dog.makeSound()//Dod barks
                dog.sleep()//Animal is sleeping
            }

**Polymorphism**

one function or object can behave in different ways depending on the situation.

The word polymorphism means:
Poly = many

Morphism = forms

So polymorphism means “many forms.

**Types of Polymorphism**

There are two main types:

Compile-time Polymorphism (Method Overloading)


Runtime Polymorphism (Method Overriding)


**Compile-Time Polymorphism (Method Overloading)**

This happens when multiple functions have the same name but different parameters.



          class Calculator {
          
              fun add(a: Int, b: Int): Int {
                  return a + b
              }
          
              fun add(a: Int, b: Int, c: Int): Int {
                  return a + b + c
              }
          }
          
          fun main() {
              val calc = Calculator()
          
              println(calc.add(5, 3))
              println(calc.add(5, 3, 2))
          }



**Runtime Polymorphism (Method Overriding)**


This happens when a child class provides a different implementation of a parent class method.


          open class Animal {
          
              open fun sound() {
                  println("Animal makes a sound")
              }
          }
          
          class Dog : Animal() {
          
              override fun sound() {
                  println("Dog barks")
              }
          }
          
          fun main() {
              val animal: Animal = Dog()
              animal.sound()
          }
