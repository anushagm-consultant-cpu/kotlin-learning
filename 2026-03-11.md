**FUNCTION OVERLOADING**

Function Overloading means creating multiple functions with the same name but different parameters (different type, number, or order of parameters) in the same class or file.

**Basic Syntax**


        fun functionName(param1: Type): ReturnType {
            // code
        }
        
        fun functionName(param1: Type, param2: Type): ReturnType {
            // code
        }


**Example**


        fun add(a: Int, b: Int): Int {
            return a + b
        }
        
        fun add(a: Int, b: Int, c: Int): Int {
            return a + b + c
        }
        
        fun main() {
            println(add(5, 10)) //15
            println(add(5, 10, 15))//30
        }



**Overloading by Parameter Type**



        fun display(num: Int) {
            println("Integer: $num")
        }
        
        fun display(text: String) {
            println("String: $text")
        }
        
        fun main() {
            display(100)
            display("Hello Kotlin")
        }



**Overloading by Number of Parameters**


    fun greet(name: String) {
        println("Hello $name")
    }
    
    fun greet(name: String, age: Int) {
        println("Hello $name, Age: $age")
    }

  

**Rules for Function Overloading**

 Function name must be same
 
 Parameters must be different in:
 
       Number of parameters
       
       Type of parameters
       
       Order of parameters


**Note:**

 Return type alone cannot differentiate functions

 
      fun sum(a: Int, b: Int): Int
      fun sum(a: Int, b: Int): Double //This will cause a compile-time error.




**Advantages of Overloading**

Improves code readability

Avoids creating many different function names

Supports compile-time polymorphism

**SCOPE**

Scope determines where you can use a variable or function in your code.

**Local Scope**

A local variable is declared inside a function or block and can only be used within that block.


          
          fun main() {
              val a = 10   // local variable
          
              if (a > 5) {
                  val b = 20   // local to this block
                  println(a)
                  println(b)
              }
          
              println(a)
              // println(b)    Error (b not visible here)
          }

          


**Global Scope**

A variable declared outside all functions is called a global variable.
It can be accessed throughout the program.


          val x = 100   // global variable
          
          fun display() {
              println(x)
          }
          
          fun main() {
              println(x)
              display()
          }

          

**Class Scope (Member Scope)**

Variables declared inside a class but outside functions are called class members.



      class Student {
      
          var name = "John"   // class scope variable
      
          fun show() {
              println(name)
          }
      }
      
      fun main() {
          val s = Student()
          s.show()
      }



**Block Scope**

Variables declared inside loops or conditional blocks are only accessible within that block.


      
      fun main() {
      
          for (i in 1..3) {
              val num = i * 2
              println(num)
          }
      
          // println(num)     Not accessible
      }

**The “Object” Construct**

An Object is a static class ,all methods and variables can be accessed without an instance.

**Singleton**–One instance of a class




        fun main() {
            if(DataAccess.connect){
                DataAccess.disconnect()
            }else{
                DataAccess.connect()
            }
            println("Database is connected:${DataAccess.connect}")
            
        }
        
        object DataAccess{
            var connect= false
            
            fun connect(){
                connect =true
                println("connected to data")
            }
            fun disconnect(){
                connect =false
                println("Disconnected to data")
            }
            
        }
