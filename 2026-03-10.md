**Open Class and Inheritance**

In Kotlin, the default rule is safety first. By default, all classes are final, meaning they are "locked" and cannot be inherited. 
To allow another class to derive from it, you must explicitly unlock it using the open keyword.

**The Basic Syntax**

To enable inheritance, you must follow these three steps:
Mark the parent class as open.
Use a colon (:) to inherit.
Initialize the parent class constructor.


**Eaxmple 1:**


          
          // This class is now "open" for inheritance
          open class Vehicle(val brand: String) {
              fun honk() {
                  println("Beep beep!")
              }
          
              // Methods are also final by default; use 'open' to allow overriding
              open fun drive() {
                  println("The vehicle is moving.")
              }
          }
          
          class ElectricCar(brand: String, val batteryRange: Int) : Vehicle(brand) {
              
              // Use the 'override' keyword to change parent behavior
              override fun drive() {
                  println("The $brand is gliding silently for $batteryRange km.")
              }
          }

**Key Rules of Inheritance:**

Single Inheritance: A class can only inherit from one parent class (unlike C++).
Constructor Passing: The child class must call the parent's constructor immediately.
Override: If you want to change a function's logic in the child class, the function in the parent must also be marked open, and the child must use the override keyword.

**INTERFACE**

**Basic Syntax**
An interface is defined using the interface keyword. Unlike classes, you don't need the open keyword; interfaces are always open for implementation.


        
        interface Powerable {
            val batteryLevel: Int   // Abstract property
            fun turnOn()            // Abstract method
            
            // Interfaces can also have default implementations!
            fun showStatus() {
                println("Battery is at $batteryLevel%")
            }
      }

**Implementing an Interface**
A class "signs" the contract by using the colon (:) and must provide the logic for all abstract members using the override keyword.



          class Laptop(override val batteryLevel: Int) : Powerable {
              override fun turnOn() {
                  println("Laptop is booting up...")
              }
          }
          
          class Flashlight(override val batteryLevel: Int) : Powerable {
              override fun turnOn() {
                  println("Light is shining!")
              }
          }


**Multiple Inheritance**

This is the biggest advantage over abstract classes. A class can only have one parent class, but it can implement many interfaces.


              class Smartphone : Powerable, Driveable, Photographable {
                  // Must override everything from all three interfaces

**how to handle a "Conflict" if two different interfaces have a method with the exact same name?**

In Kotlin, when you implement multiple interfaces that happen to have a method with the same name and signature, you run into a Conflict. The compiler won't know which one to use, so it forces you to resolve it manually.
This is often called the "Diamond Problem" in other languages, but Kotlin handles it very explicitly.



        interface Camera {
            fun start() {
                println("Lens opening... Camera ready.")
            }
        }
        
        interface Phone {
            fun start() {
                println("Screen lighting up... Phone ready.")
            }
        }


**If you create a Smartphone class, the compiler will get confused:**

        
        class Smartphone : Camera, Phone {
            // ERROR: Class 'Smartphone' must override 'start' because it 
            // inherits multiple implementations of it.
            
        }


**How to Resolve the Conflict**

To fix this, you must override the conflicting method in your class. You can then choose to:
Write entirely new logic.
Call one specific interface's version.
Call both versions.
You use the super<InterfaceName> syntax to specify which parent you are talking to.



        class Smartphone : Camera, Phone {
            
            override fun start() {
                // Resolve the conflict
                super<Camera>.start() // Call Camera's start
                super<Phone>.start()  // Call Phone's start
                
                println("Smartphone is fully initialized.")
            }
        }
        
        fun main() {
            val myDevice = Smartphone()
            myDevice.start()
        }




**Summary Checklist for Conflicts**
Identify: The compiler highlights the class name in red.
Override: You must override the conflicting method name.
Delegate: Use super<ParentName>.method() to reuse existing code from the i
