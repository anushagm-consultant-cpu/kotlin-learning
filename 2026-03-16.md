**COLLECTION**

Collections are used to store and manage groups of objects (elements)

**Types of Collections**

**1.List**

Ordered collection

Allows duplicate elements

Elements accessed using index

**Types of List in Kotlin**

1.Read-Only List

Cannot add, remove, or modify elements.

Created using listOf().


        fun main(){
        val fruits = listOf("Apple", "Banana", "Orange")
        
        println(fruits)//Apple, Banana, Orange
        println(fruits[1])//Banana
        }

2. Mutable List

Elements can be added, removed, or updated.

Created using mutableListOf().

          
          fun main(){
          val fruits = arrayListOf("Apple", "Banana")
          
          fruits.add("Orange")
          fruits.remove("Banana")
          
          println(fruits)//[Apple, Orange]
          }


| Function     | Description                |
| ------------ | -------------------------- |
| `add()`      | Adds element               |
| `remove()`   | Removes element            |
| `get()`      | Gets element by index      |
| `size`       | Returns number of elements |
| `contains()` | Checks element exists      |



      


**Accessing List Elements**

        fun main(){
        val list = listOf("A","B","C")
        println(list[2])
        for (item in list) {
            println(item)
        }
        }


**2.Set**


A Set is a collection that stores unique elements.

Unordered collection

No duplicate elements

      fun main(){
      val numbers = setOf(1, 2, 3, 4)
      println(numbers)
      }

**Types of Set**

**1. Read-Only Set**
   


Cannot add or remove elements

Created using setOf()

**Example:**

fun main(){
      val fruits = setOf("Apple", "Banana", "Orange")
      println(fruits)
      }


**2. Mutable Set**

Elements can be added or removed

Created using hashSetOf()

**Example:**


fun main(){
          val fruits = hashSetOf("Apple", "Banana")
          
          fruits.add("Orange")
          fruits.remove("Banana")
          
          println(fruits)
          }


        fun main(){
        val numbers = setOf(1,2,3,3,4)
        
        println(numbers)//[1,2,3,4]
        }


| Function     | Purpose              |
| ------------ | -------------------- |
| `add()`      | Add element          |
| `remove()`   | Remove element       |
| `contains()` | Check element exists |
| `size`       | Number of elements   |




    fun main(){
              val nums = hashSetOf(1,2,3)
              
              nums.add(4)
              println(nums.contains(2))
              println(nums.size)
              }


**3.Map**

Stores key–value pairs

Keys must be unique

A value can be duplicated

Used to store related data

Example:


          fun main() {
             val student = mapOf(
              "name" to "John",
              "age" to 20
          )
          
          println(student)

          println(student["name"])//john
          

**hashMapOf()**

          fun main() {
             val marks = hashMapOf<String, Int>()
          
          marks["Math"] = 90
          marks["Science"] = 85
          
          println(marks)
          }


**Iterating a Map**

            fun main(){
            val marks = mapOf(
                "Math" to 90,
                "Science" to 80
            )
            
            for ((key, value) in marks) {
                println("$key = $value")//Math = 90
Science = 80
            }
}



| Function          | Purpose            |
| ----------------- | ------------------ |
| `put()` / `[]=`   | Add element        |
| `remove()`        | Remove element     |
| `containsKey()`   | Check key exists   |
| `containsValue()` | Check value exists |
| `size`            | Number of pairs    |




**Lambdas**

Lambdas in Kotlin are anonymous functions—functions without a name that you can pass as values to other functions. They’re commonly used for short pieces of behavior, especially with collections, callbacks, and functional-style programming.


      { parameters -> body }


      Example:

      
            fun main(){
            val sum = { a: Int, b: Int -> a + b }
      println(sum(3, 4))  // 7
            }

  **Lambda as Function Parameter**
    
        fun operate(a: Int, b: Int, op: (Int, Int) -> Int): Int {
            return op(a, b)
        }
        
        fun main(){
        val result = operate(4, 3) { x, y -> x + y }
        println(result) // 7
        }


  **Lambdas with Collections**

          val numbers = listOf(1, 2, 3, 4)
        
        val doubled = numbers.map { it * 2 }
        
        println(doubled) // [2,4,6,8]


**Lambda with Multiple Statements**

      val greet = { name: String ->
          println("Hello")
          "Welcome $name"
      }
      
      println(greet("Alex"))


 **Lambda vs Function**

Lambda:

      val square = { x: Int -> x * x }

Function:

        fun square(x: Int): Int {
            return x * x
        }


Use lambdas for small operations and functions for larger logic.

**Example**

        data class User(val name: String, val age: Int)
        
        val users = listOf(
            User("Alice", 25),
            User("Bob", 20)
        )
        fun main(){
        val sorted = users.sortedBy { it.age }
        }
