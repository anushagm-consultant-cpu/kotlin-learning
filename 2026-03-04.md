
**NULL SAFETY**

In Kotlin, types are non-nullable by default. If you try to assign null to a standard variable, the code won't even compile. To allow a variable to hold null, you must explicitly mark it with a ?.

Non-Nullable: var name: String = "Anusha" (Cannot be null)

Nullable: var name: String? = null (Can be null)

Check for null with the if conditional

When working with nullable types, you need to handle nullability safely to avoid an NPE. One way to handle this is checking for nullability explicitly with the if conditional expression.

For example, check whether b is null and then access b.length:



            val b: String? = null
            // Checks for nullability first and then accesses length
            val l = if (b != null) b.length else -1
            print(l)  // -1


            val b: String? = "Kotlin"
            // Checks for nullability first and then accesses length
            if (b != null && b.length > 0) {
                print("String of length ${b.length}")
                // String of length 6
            } else {
                // Provides alternative if the condition is not met
                print("Empty string")
            }


**Safe Calls (?.)**

When dealing with a nullable variable, you can't just access its properties directly. The Safe Call operator allows you to access a property only if the reference is not null. If it is null, the entire expression returns null instead of crashing.


                fun main(){
                var n:String?="code"
                    n =null
                    println(n)//null
                
                    var name:String?="code"
                    name =null
                    println(name?.length)//null
                // If name is null, length becomes null. No crash!
                }



**The Elvis Operator (?:)**

The Elvis operator  allows you to provide a default value if the expression on the left is null.


            fun main(){
            val b:String?=null
            val l:Int=if(b!=null) b.length else 0
            }

            
            Instead of writing the complete if expression , we can handle this in a more concise way with the Elvis operator ?:

            
            fun main(){
            val b:String?=null
            val l=b?.length?:0
            println(l)  //0
            }


            fun main(){
            var age:Int?=null
                println( age?: "Invalid age" )//Elwis operator
            val length: Int = name?.length ?: 0
            // If name is null, return 0 instead of a null value.
            }

**Not-Null Assertion(!!)**


It forcibly converts any value to a non-nullable type. Use this sparingly. If the variable turns out to be null, it will throw a NullPointerException immediately.


            var company:String?="newscorp"
                println(company!!.length)//Non null assertion
            // High risk: will crash if company is null



Note:The !! operator is particularly useful when you are confident that a value is not null and there's no chance of getting an NPE, but the compiler cannot guarantee this due to certain rules. In such cases, you can use the !! operator to explicitly tell the compiler that the value is not null.
