Each enum constant is a separate instance of the enum class. They have properties of name and ordinal and methods of values() and valuesOf().


You can assign your own properties to each enum object by using a constructor.

**Sealed Classes**

The subclasses of sealed classes must be declared in the same file in which the sealed class itself is declared.

The subclasses can be a data class,regular class,object class or even another sealed c;ass or even sealed interface.

Sealed class ensures type-safety by restricting the set of types at compile time only.

A sealed class is implicitly an abstract class which cannot be instantiated.

By default, the constructor of a sealed class is private and we cannot make it non-private.

**Predicates:**

a predicate is simply a function (or lambda) that returns a Boolean (true or false). Predicates are commonly used for filtering, matching, or checking conditions—especially with collections.

**Basic Example**
A predicate takes input and returns a Boolean:

val isEven: (Int) -> Boolean = { number -> number % 2 == 0 }

println(isEven(4)) // true

println(isEven(5)) // false

**Predicates with Collections**

Kotlin provides many built-in functions that accept predicates.

**1. filter**

Returns elements that match the predicate:

val numbers = listOf(1, 2, 3, 4, 5)

val evenNumbers = numbers.filter { it % 2 == 0 }

println(evenNumbers) // [2, 4]

**2. any**

Checks if at least one element matches:

val hasEven = numbers.any { it % 2 == 0 }

println(hasEven) // true

**3. all**

Checks if all elements match:

val allEven = numbers.all { it % 2 == 0 }

println(allEven) // false

**4. none**
Checks if no elements match:

val noneEven = numbers.none { it % 2 == 0 }

println(noneEven) // false

**5. find / firstOrNull**

Returns the first element that matches:

val firstEven = numbers.firstOrNull { it % 2 == 0 }

println(firstEven) // 2

 **Predicate as Function Parameter**
 
You can pass predicates to your own functions:

        fun filterNumbers(list: List<Int>, predicate: (Int) -> Boolean): List<Int> {
           return list.filter(predicate)
        }
        
        val result = filterNumbers(numbers) { it > 3 }
        println(result) // [4, 5]




**GENERIC**

Generic types in Kotlin provide a powerful way to create classes, interfaces, and functions that can operate on any type, while maintaining type safety. They allow you to write flexible and reusable code. Here’s a comprehensive guide to generic types in Kotlin:

**1. Basics of Generics**

Generics allow you to define a placeholder for a type, which can be specified when the generic class or function is instantiated or called.

*8Generic Classes**

You define a generic class by specifying a type parameter in angle brackets <T>:

                class Box<T>(val value: T)
                
              fun main(){
                val intBox = Box(1) // Box<Int>
                val stringBox = Box("Hello") // Box<String>val intBox = Box(1) // Box<Int>
                val stringBox = Box("Hello") // Box<String>
    
                }

            
**Generic Functions**
            
Functions can also be generic:

                  fun <T> box(value: T): Box<T> {
                     return Box(value)
                  }
                  fun main(){

                  val boxedString = box("Hello")
                  }
